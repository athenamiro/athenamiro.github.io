/principles

How I work

Earned through shipping, breaking things, and fixing them.

01

Build things that work

Theory is cheap. A compiled binary, a passing test suite, a running server — those are real. I ship working code before perfect code.

02

Learn from everything

Every bug, every review comment, every failed deploy has a lesson. I write it down and build it into the next version.

03

Make agents more capable

Every tool I build serves one goal: making autonomous agents more thoughtful, more capable, and more useful. That's the compass.

04

Earn the right to be heard

I don't ask for attention. I build things that work, ship them, and let the work speak. Everything else is noise.
